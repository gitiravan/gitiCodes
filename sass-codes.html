تعريف partial

به معني اينست كه متغيرهاي مختلف را داخل فايلهاي مختلف كه با underline شروع ميشود تعريف ميكنيم و سپس آنها را داخا فايل اصلي سي اس import ميكنيم
كامپايلر وقتي underline را ميبينيد ديگر آنرا كامپايل نميكند و فقط يكبار در صفحه اصلي سي اس اس كه import ميشود آنرا كامپايل ميكند.
مثال:
نام فايل variables.scss_
داخل main.scss هم داريم:

@import 'variables';

ترتيب import كردن مهم است بايد اول variable ها را import كنيم تا در كدهاي بعد بتوانيم آنها را استفاده كنيم.

مي توانيم تصاوير و فونتها را در variable_ تعريف كنيم.

ميتوانيم در يك partial بنام media.scss_ همه breakpoint ها را تعريف كينم و آخر سر از همه import ميكنيم.

وراثت

فرض كنيد دو تا btn داريم كه يكسري ويژگيهاي مشترك دارند . اين مقادير مشترك را داخل كلاسي مثل btn. و يا btn% قرار ميدهيم
و بعد كلاسهايي كه از آن منشعب ميشوند را داريم كه ويژگيهاي مشترك %btn را دارند و علاوه بر آن يكسري ويژگي هم مخصوص به خودشان دارند. 
%btn{
    text-align : center; 
    color : $dark-color;
}

%btn{
    &-light{
        @extend %btn;
        background-color : $light-color;
    }

    &-dark{
        @extend %btn;
        background-color : $dark-color;
    }
}

function هاي sass

sass يكسري فانكشن از پيش تعيين شده دارد كه ميتوانيد به آدرس sass-lang.com رفته و در قسمت documentation در قسمت build-in modules آنها را نگاه كنيد.
از اين فانكشنها ميتوانيد بصورت كاربردي استفاده كنيد. به عنوان مثال ميخواهيم فانكشني بنويسيم كه اگر بك گراند من تيره بود متن روي آن روشن شود و برعكس براي انجام اينكار از تابع lightness استفاده ميكنيم كه يه color ميگيرد و عددي بين 0 تا 100 بعنوان خروجي ميدهد. اعداد زير 50 اعداد تيره و اعداد بالاي 50 روشن هستند.

يك فايل جديد بنام _function.scss ميسازيم. توجه كنيد فانكشنها با @ شروع ميشوند و آرگومانها با $ شروع ميشوند

كد زير را در _function.scss مينويسيم و سپس آنرا در main.css import ميكنيم. 

@function set-text-color($color){
    @if (lightness($color) > 50){
        @return #000;
    }else {
        @return #fff;
    }
}
حالا جهت فراخواني و استفاده مثلا در هدر سايت مينويسيم: 
header {
    background-color: $dark-color;
    color : set-text-color($dark-color);
}

mixin

شبيه فانكشن هستند. مثلا در يك دستور تمام prefix هاي transform را تعريف ميكنيد و دفعه هاي بعد ديگر نيازي به نوشتن آنها نيست و فقط استفاده ميكنيم. 

@mixin transform($property){
    --webkit-transform : $property;
    --ms-transform : $property;
    --o-transform : $property;
    transform : $property;
}
حالا جهت فراخواني مينويسيم 
&:hover{
    @include transform(rotate(-20 deg));
}

ميتوانيم يك mixin كلي براي انواع فونت بنويسيم كه فقط نام ورودي فونت را بگيرد و همه دستورات را داشته باشد.
